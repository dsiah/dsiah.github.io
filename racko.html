The Rules of Rack-O

About a year ago my friends and I discovered a weird card game called Rack-O. Each player has a hand of cards that they must sort. The catch is that they can only move one card per turn, in one of two ways:

  1. Take a card from the deck, swapping it with one card from their hand, and placing the old card in the discard pile.
  2. Take the top card from the discard pile, swap it with one card from their hand, and place the old card in the discard pile face up.

Order can be established in any sort of way but traditionally the players each take a card, the lowest card designates the dealer and the player to the left of the dealer starts. The game starts with one card being turned over onto the discard pile.

Design Decisions

I have begun prototyping the game engine in python, it's a relatively simple concept that translates well to Object Oriented structure. For example a Deck has Cards and a Player has a hand which has Cards. The real fun comes with implementing this game in real time. I would like to take this game engine and place it at the heart of an asynchronous web server that can host a game and broadcast changes. Right now the greatest challenge is how I will prevent players from inspecting the state of other players hands. With a web server I can distribute each hand to the client preventing them from seeing the entire state of the game, but I have decided to hold off on making this decision until I complete some research on distributed games.

Ideally I would like this game to be simultaneously mobile and browser based. Most of the inspiration comes from what I like to call "distributed games" in which people can connect to each other's devices and play between a network. My favorite example in recent memory is a game called Space Team. But the main hinderance is that a connection mustbe opened between client and server instead of a simple network from device to device.

Currently I am thinking of using a framework like socket.io to open socket connections to a server which would distribute the game information appropriately but there may be a more elegant solution...
